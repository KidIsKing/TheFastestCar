# TheFastestCar - Игра на Pygame

### **Бесконечная езда по шоссе с видом сверху.**

![Фото с демкой игры](images/TheFastetsCar_demo.jpg)

## Описание

- Суть игры заключается управлении машиной на бесконечном шоссе в две полосы в одну сторону, две в другую.
- Главной задачей является максимально долгое время выживания на дороге для заработка наибольшего количества очков или денег ради покупки новых автомобилей (планируется добавить целый автопарк, возможно, с улучшениями).
- Врагами будут являться машины, едущие равномерно в едином направлении с игроком или обратном с разной скоростью.
- Важнейшую роль в оценке игры играет использование сложных или не очень алгоритмов в коде. Планируемое применение их:
    1.  сложное появление врагов;
    2.  постепенное увеличение скорости игрока (то есть игрок двигается с ускорением - const?) и врагов, которые с каждой минутой (или другим заданным значением) будут появляться с всё более большей скоростью;
    3.  модель повреждения и отнимаемое количество здоровья при столкновении;
    4.  случайная генерация бонусов на дороге (аля Fastlane) ("Случайность и вероятность");
    5.  определение меры улучшения при прокачке авто игрока (опционально, зависит от добавления автопарка).
    6.  изменение монёвренности и других характеристик автомобилей при перемене погодных условий (опционально - "Системы частиц")
    7.  смена дня и ночи (опционально - "Простое освещение")
    8.  поведение врагов должно быть случайно, но с минимальными рисками для врага - перестроения, обгоны и т.д. (поведение врагов и как они думают - "Минимакс и деревья игр", "Генетические алгоритмы", очень опционально - "Нейросети для игр")
    9.  админ версия при аутентификации (опционально - "Отладка и профилирование")

    ## Алгоритм случайного спавна врагов

    Для появления врагов в игре используется не просто `random()`, а связка из двух идей из темы "Случайность и вероятность":

    1.  **Взвешенный выбор** - по мере роста скорости игрока уменьшается интервал между спавнами и растёт вероятность появления группы из 2-3 машин вместо одной.
    2.  **Shuffle bag** - координаты спавна берутся из мешка с заранее подготовленными позициями по дороге. После того как все позиции были использованы, мешок перемешивается заново.

    Тот же подход используется и для бонусов: у них есть свой таймер, свои случайные позиции и свой shuffle bag, поэтому при росте скорости бонусов становится больше, но спавн остаётся честным и непредсказуемым.

    Такой подход даёт нужный эффект для игры:

    - на медленной скорости врагов меньше;
    - на высокой скорости врагов становится заметно больше;
    - позиции не повторяются слишком часто подряд, поэтому спавн выглядит честнее и живее;
    - при этом остаётся элемент случайности, который удобно объяснить на экзамене.

    В коде также используются отдельные генераторы случайностей для разных подсистем, чтобы поведение врагов не зависело от будущих случайных событий в других частях игры. Это соответствует идее из материала про seed и независимые Random.

## Используемые технологии

- python 3.10.11
- PyGame 2.6.1

## Запуск

1\. Создать виртуальное окружение:

```bash
python -m venv venv
```

2\. Активировать его:

```bash
source venv/Scripts/activate
```

3\. Установить зависимости:

```bash
pip install -r requirements.txt
```

4\. Обновить версию `pip` (опционально):

```bash
python.exe -m pip install --upgrade pip
```
