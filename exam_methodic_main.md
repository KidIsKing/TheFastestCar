Эта методичка написана по **текущему `src/main.py`** и ориентирована на устную защиту: что где находится, какие алгоритмы используются, почему они выбраны и как это объяснить преподавателю простыми словами.

## 1. Что это за проект

**TheFastestCar** — бесконечная аркада на шоссе с видом сверху. Игрок управляет машиной, уворачивается от врагов, собирает бонусы, зарабатывает очки и выживает как можно дольше.

Главная идея проекта для экзамена не просто в том, чтобы сделать игру, а в том, чтобы показать, что в ней есть **несколько разных алгоритмов**, которые можно объяснить и защитить:

- взвешенный случайный выбор;
- shuffle bag;
- AABB-проверка столкновений через hitbox;
- гауссово распределение для урона и лечения;
- генетический алгоритм для поведения врагов;
- сглаживание движения и изменения скорости;
- независимые генераторы случайных чисел для разных подсистем.

## 4. Алгоритм взвешенного выбора

### Что делает `WeightedChoiceTable`

Класс `WeightedChoiceTable` реализует **взвешенный случайный выбор**. Это значит, что элементы имеют разные вероятности выпадения.

Пример из игры:

- на низкой скорости чаще спавнится 1 враг;
- на высокой скорости увеличивается шанс группы из 2-3 врагов.

### Как это работает

Идея такая:

1. Каждому варианту назначается вес.
2. Суммируются накопленные веса.
3. Генерируется случайное число в диапазоне от 0 до суммы весов.
4. Через бинарный поиск определяется, какой элемент выпал.

### Почему используется `bisect`

`bisect_left` ищет позицию за $O(log n)$, а не перебором всех элементов.

Это хороший аргумент на экзамене:

> Здесь применяется не просто случайный выбор, а эффективный взвешенный выбор с бинарным поиском, поэтому метод работает быстро даже при росте количества вариантов.

### Мини-пример

```python
table = WeightedChoiceTable(random.Random())
table.add(1, 70)
table.add(2, 20)
table.add(3, 10)
result = table.roll()
```

В этом случае единица выпадает чаще всего.

### Как объяснить преподавателю

> Это алгоритм для ситуации, где нам нужна не равная вероятность, а управляемое распределение. Например, одна машина должна появляться чаще, чем группа из трёх машин.

## 5. Алгоритм shuffle bag

### Что делает `ShuffleBag`

`ShuffleBag` — это мешок элементов без повторов до полного исчерпания списка.

В проекте он используется для координат спавна по полосам.

### Зачем он нужен

Если просто брать случайную координату, может получиться, что одна и та же полоса будет выпадать слишком часто подряд. Это выглядит неестественно.

`ShuffleBag` решает это так:

1. Берёт все возможные позиции.
2. Перемешивает их.
3. Выдаёт по одной.
4. Когда список заканчивается, перемешивает заново.

### Пример

```python
bag = ShuffleBag(random.Random(), [100, 200, 300])
print(bag.draw())
print(bag.draw())
print(bag.draw())
```

Сначала выпадут все позиции по одному разу в случайном порядке, а потом мешок начнёт новый цикл.

### Как объяснить на защите

> Shuffle bag позволяет избежать нежелательных повторов и делает спавн визуально честнее. Это не чисто случайная генерация, а случайность с контролем распределения.

## 6. Генетический алгоритм

Это один из самых важных блоков для экзамена.

### Что он делает в игре

`EnemyGenome` хранит параметры поведения врага:

- `aggressiveness` — агрессивность;
- `burst_rate` — частота смены поведения;
- `drift_intensity` — сила бокового движения.

`GeneticAlgorithm` создаёт популяцию таких геномов, оценивает их, отбирает лучших, скрещивает и мутирует.

### Почему это именно генетический алгоритм

Потому что есть все основные части ГА:

1. **популяция**;
2. **fitness**;
3. **отбор**;
4. **скрещивание**;
5. **мутация**;
6. **элитизм**.

### Кодовая идея

```python
self.population = [EnemyGenome() for _ in range(population_size)]
```

Это стартовая популяция.

Дальше:

```python
sorted_pop = sorted(self.population, key=lambda g: g.fitness, reverse=True)
new_population = [g.copy() for g in sorted_pop[:elitism]]
```

Это элитизм: лучший геном сохраняется.

### Как работает fitness

Сейчас fitness в проекте эвристический. Это значит, что мы не запускаем полноценную симуляцию боя внутри ГА, а оцениваем геном по тому, насколько его параметры выглядят сбалансированно и активно.

Пример логики:

```python
aggr_score = 100 - abs(aggressiveness - 0.8) * 60
behavior_score = (burst_rate * 50) + (drift * 60)
fitness = max(0, aggr_score + behavior_score)
```

### Почему такой fitness хороший для учебного проекта

Потому что он:

- понятный;
- легко объясняется;
- не требует тяжёлой симуляции;
- даёт управляемый результат.

### Турнирный отбор

```python
tournament = random.sample(self.population, min(tournament_size, len(self.population)))
return max(tournament, key=lambda g: g.fitness)
```

Смысл:

- берём несколько случайных особей;
- из них выбираем лучшую;
- так сохраняется баланс между случайностью и качеством.

### Скрещивание

```python
point = random.randint(1, len(parent_a.genes) - 1)
child_genes = parent_a.genes[:point] + parent_b.genes[point:]
```

Это одноточечное скрещивание.

Объяснение простыми словами:

> Ребёнок получает часть признаков от одного родителя и часть от другого.

### Мутация

```python
mutation = random.gauss(0, strength * gene)
mutated_gene = max(0.0, min(2.0, gene + mutation))
```

Тут используется гауссово распределение.

Это означает:

- изменения обычно маленькие;
- большие отклонения редки;
- хорошие решения не ломаются резко.

### Как это выглядит в самой игре

В `Game` создаётся объект GA:

```python
self.ga = GeneticAlgorithm(population_size=8, mutation_rate=0.35)
self.ga.evaluate_population()
self.best_enemy_genome = self.ga.get_best_genome()
```

Потом при спавне врага используется лучший текущий геном:

```python
Enemy(..., genome=self.best_enemy_genome)
```

И раз в несколько спавнов происходит эволюция:

```python
if self.enemy_spawn_count % self.evolution_interval == 0:
    self.ga.evaluate_population()
    self.ga.evolve()
    self.best_enemy_genome = self.ga.get_best_genome()
```

### Важный тезис для защиты

> Это не нейросеть, а эволюционный алгоритм. Он не учится “в процессе игры”, а подбирает параметры поведения по принципам наследования и отбора.

## 8. Игрок

Класс `Player` хранит:

- изображение машины;
- `rect`;
- здоровье;
- хитбокс;
- вертикальную плавную анимацию движения.

### Почему хитбокс меньше спрайта

Если сравнивать целый спрайт, столкновения будут слишком жёсткими. Поэтому хитбокс уменьшен:

```python
hb_w = int(self.rect.width * 0.7)
hb_h = int(self.rect.height * 0.6)
```

Это делает игру честнее.

### Как движется игрок

По горизонтали игрок двигается с клавишами:

```python
if keys[pygame.K_LEFT] or keys[pygame.K_a]:
    self.rect.x -= 10
if keys[pygame.K_RIGHT] or keys[pygame.K_d]:
    self.rect.x += 10
```

По вертикали есть плавное смещение, которое создаёт визуальный эффект ускорения:

```python
self.current_y += (target_y - self.current_y) * smoothing
```

### Что сказать на защите

> У игрока используется не только обычное движение, но и сглаживание положения. Это делает движение более естественным, чем резкий скачок координат.

## 9. Враг

Класс `Enemy` — это основной объект, где видно применение ГА.

### Какие параметры у врага

Враг использует геном и извлекает из него поведение:

```python
self.aggressiveness = self.genome.genes[0]
self.burst_rate = self.genome.genes[1]
self.drift_intensity = self.genome.genes[2]
```

### Что означает каждый параметр

- `aggressiveness` — влияет на урон и на силу поведения;
- `burst_rate` — как часто враг меняет режим движения;
- `drift_intensity` — насколько сильно он уходит в сторону.

### Почему поведение стало заметнее

Потому что враг теперь:

- меняет скорость вперёд/назад;
- перестраивается быстрее;
- сильнее отклоняется в стороны;
- реагирует на границы дороги.

### Как устроено движение

Враг хранит:

- `base_speed`;
- `speed_variation`;
- `speed_target`;
- `drift_speed`;
- `drift_target_speed`.

То есть движение состоит из двух частей:

1. вертикальная скорость;
2. боковой дрифт.

### Вертикальная скорость

```python
if now >= self.next_speed_change:
    self.speed_target = self.rng.uniform(-0.3, 0.3) * self.base_speed * self.aggressiveness
```

Это создаёт эффект ускорения и замедления.

Затем скорость не меняется резко, а сглаживается:

```python
self.speed_variation += (self.speed_target - self.speed_variation) * 0.12
```

### Боковой дрифт

```python
if now >= self.next_drift_change:
    self.drift_target_speed = self.rng.choice([-1, 1]) * self.rng.uniform(
        0.4 * self.drift_intensity, 1.0 * self.drift_intensity
    )
```

Потом он тоже сглаживается:

```python
self.drift_speed += (self.drift_target_speed - self.drift_speed) * 0.25
```

### Почему это важно

Это позволяет объяснить, что враг не просто “едет вниз”, а ведёт себя как искусственный агент с параметризованным поведением.

### Урон врага

Урон считается через гауссово распределение:

```python
mean = float(self.base_damage)
std = max(1.0, mean * 0.15)
dmg = int(round(self.rng.gauss(mean, std)))
```

Это означает:

- урон колеблется вокруг среднего;
- значения не слишком хаотичны;
- иногда враг бьёт чуть слабее или сильнее.

### Как это объяснить на защите

> Враг управляется не случайным набором действий, а параметрами, которые могут эволюционировать. Поэтому его поведение можно считать алгоритмически заданным и адаптивным.

## 11. Дорога

`Road` реализует бесконечный скроллинг.

### Как это работает

Две копии текстуры дороги двигаются вниз:

```python
self.y1 += move
self.y2 += move
```

Когда одна уходит за экран, она переносится наверх.

### Почему это хорошо

Это создаёт иллюзию бесконечного движения, хотя на самом деле используется всего две текстуры.

### Как сказать на защите

> Здесь используется простой алгоритм зацикливания текстур, который даёт бесконечный фон без больших затрат ресурсов.

## 12. Главный класс Game

Это центр всей программы.

### Что он делает

`Game` управляет:

- инициализацией PyGame;
- генераторами случайных чисел;
- спавном врагов и бонусов;
- скоростью;
- коллизиями;
- очками;
- здоровьем;
- отрисовкой.

### Почему здесь несколько RNG

Очень важный момент для экзамена.

В проекте есть:

```python
self.master_seed = random.randrange(2**32)
self.master_rng = random.Random(self.master_seed)
self.spawn_rng = random.Random(self.master_rng.randrange(2**32))
self.enemy_rng = random.Random(self.master_rng.randrange(2**32))
self.bonus_rng = random.Random(self.master_rng.randrange(2**32))
```

Это значит:

- спавн врагов не влияет на поведение врагов;
- враги не влияют на бонусы;
- всё можно сделать детерминированным при одном seed.

### Что говорить

> Я разделила случайность на независимые подсистемы. Это удобно и для отладки, и для демонстрации алгоритмов случайности.

## 14. Скорость игрока и сглаживание

В `update()` есть важная логика движения.

### Почему это не просто `speed += 1`

Потому что тогда движение будет дёрганым. Здесь используется сглаживание:

```python
self.player_speed += (target_speed - self.player_speed) * self.speed_smoothing
```

### Что это даёт

- плавный разгон;
- плавное торможение;
- отсутствие резких скачков.

### Вертикальный эффект

```python
lift_amount = self.max_lift * (self.throttle**2)
```

Это создаёт ощущение, что машина чуть приподнимается при ускорении.

### Как объяснить

> Это не физический симулятор, а визуально-игровое сглаживание. Оно повышает ощущение скорости и делает управление приятнее.

## 15. Проверка столкновений

### Почему используется hitbox

Полный спрайт не всегда подходит для коллизий. Он часто слишком большой и визуально нечестный.

Поэтому используются уменьшенные прямоугольники:

- у игрока;
- у врага;
- у бонуса.

### Тип проверки

Это **AABB** — Axis-Aligned Bounding Box.

### Суть

Если два прямоугольника пересекаются, значит произошло столкновение.

```python
if self.player.hitbox.colliderect(enemy.hitbox):
    return enemy
```

### Как объяснить

> Это классический и очень быстрый метод проверки столкновений для 2D-игр.

### Почему это хорошо для экзамена

Потому что алгоритм простой, понятный и реально используется в игре.

- используется hitbox, а не весь спрайт;
- это AABB;
- так честнее и быстрее.

### Про движение
- скорость сглаживается;
- фон зациклен;
- враги двигаются не только вниз, но и по горизонтали.

## 18. Возможные вопросы на экзамене и готовые ответы

### Вопрос 1
**Почему вы выбрали генетический алгоритм?**

Ответ:
> Потому что он хорошо подходит для подбора параметров поведения врагов. В игре нужны не точные решения, а разнообразные и правдоподобные шаблоны поведения. ГА позволяет получить это через отбор, скрещивание и мутацию.

### Вопрос 2
**Почему не нейросеть?**

Ответ:
> Нейросеть сложнее объяснить на защите, требует данных и обучения. Для экзамена генетический алгоритм лучше: он наглядный, простой и сразу показывает применение алгоритмов из курса.

### Вопрос 3
**Почему фитнес эвристический, а не через полную симуляцию?**

Ответ:
> Потому что это учебный проект, и полная симуляция увеличила бы сложность и время работы. Эвристика даёт управляемую и объяснимую оценку параметров.

### Вопрос 4
**Что делает shuffle bag?**

Ответ:
> Он выдаёт позиции без повторений, пока не закончится весь набор, а потом перемешивает заново. Это делает спавн более честным.

### Вопрос 5
**Зачем нужен weighted choice?**

Ответ:
> Чтобы разные события происходили с разной вероятностью. Например, одна машина может появляться чаще, чем группа из трёх.

### Вопрос 6
**Зачем хитбоксы меньше спрайтов?**

Ответ:
> Чтобы столкновения были честнее. Полный спрайт обычно слишком большой и создаёт ощущение, что игрок погибает от касания краем.

### Вопрос 7
**Почему используется гауссово распределение?**

Ответ:
> Потому что оно даёт значения вокруг среднего с редкими отклонениями. Это удобно для урона и лечения: результат предсказуемый, но не абсолютно одинаковый.

### Вопрос 8
**Как случайность изолирована в проекте?**

Ответ:
> У меня есть отдельные генераторы для спавна, врагов и бонусов. Это делает поведение независимым и облегчает отладку.
